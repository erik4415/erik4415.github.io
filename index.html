About

Hello to one and all. My name is Erik Hough, in case you didn't notice from the gigantic banner above, and I'm a software developer. I graduated from the University of Minnesota in 2013 and started working as a black box software tester for a healthcare company called Epic. I quickly started to learn as much about programming as I could, and quickly found a passion for all things software. I took some classes offered by our company to learn more about coding, and then started taking computer science classes through the University of Wisconsin - Madison. Within my first year at Epic I transitioned from doing black box QA, to writing set up code for unit tests, along with the actual unit tests, and server side testing scripts. Soon I was in charge of all automated testing for our team and was helping other teams implement their own automated testing. Along with this I also did a lot of development to fix existing bugs, and developed a bunch of new tools to help internal black box QA'ers test more efficiently. Besides my work experience I have also developed quite a few of my own personal projects, just for fun. I haven't had the most traditional path to becoming a software developer, but fortunately I have had some great opportunities and a passion for making cool stuff. In my free time I love to hike and generally be active outdoors, I also love to cook and spend time with my dogs. Your probably sick of hearing about me so take a look at my projects and resume below, hope you enjoy!

Personal Projects

Find a Good Neighborhood Website

When I moved to Madison, WI, it was difficult to decide where to live. I created this website using the Google maps API’s. It searches based on surrounding landmarks and shades area based on their rating. Good areas include coffee shops, restaurants, parks, and other similar data points nearby. Bad areas include fast food restaurants and hospitals. The map actually does a pretty good job of correlating with crime statistics for cities. I got a lot of experience of working with complex API’s and figuring out how to implement a complex web project.

Audiogram Website

One of the main components of an Audiologists workflow is to create an audiogram, which is a way to graph the hearing levels of a patient. Audiograms usually contain a lot of information, so I wanted to create a simple design where the most important information stands out and gives an Audiologist the ability to highlight different tests. I got a lot of practice with JavaScript and working on a complex interface.

Small Business Website

This is a website I made for a local contractor. He wanted people to be able to his business on a web search and have a good portfolio of his work. This is a pretty simple brochure website but I learned a lot about deploying a website and getting a site indexed for major search engines.

Noteworthy Academic Projects

Word Cloud Generator - I created this in Java for a Data Structures class project. The point of the project was to implement specific data structures, so a combination of a BST and priority queue was used. I learned a lot about implementing interfaces, data structures, and common design patterns.

Custom Programming Language Compiler - My own programming language created for my Compilers and Programming Languages class. I Created the symbol table, parser, type checker, and code generator. Uses a CUP parser, the jvm for code generation, and jasmine for compilation. Written in Java.
